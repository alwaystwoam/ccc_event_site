// 11:30 — THE REFRAME
What does an LLM actually do?
It predicts the next most likely token.
It doesn't think. It doesn't understand. It pattern-matches at massive scale.
// HOW LLMS WORK
Token prediction
Every response is generated one token at a time. The model picks the most probable next token given everything before it.
This means:
• It can be confidently wrong (hallucinations)
• It's sensitive to how you phrase things
• The same prompt can produce different outputs
• It has no memory between conversations
Understanding this changes how you work with it. You stop treating it like a search engine and start treating it like a probabilistic pattern engine.
// CONTEXT WINDOWS
Context management
The model can only "see" what's in the context window.
Everything outside the window doesn't exist to the model.
  • Long conversations fill up the window
  • Early instructions get pushed out
  • Quality degrades as context grows
  • The model starts "forgetting" your constraints
How to stay in control
  • Start fresh often — new conversation for new tasks
  • Feed relevant files — give it only what it needs
  • Keep specs tight — concise > comprehensive
  • Reference, don't repeat — point to files instead of pasting

// 12:00 — AI MODES
Three ways to work with AI.
Each mode has a different purpose. The order matters.
// PLAN → ASK → AGENT
The three modes
01
Plan
Think before you build. The AI helps you design, outline, and pressure-test your idea. No code is written yet.
02
Ask
Explore and learn. Ask questions, get explanations, understand concepts. Read-only — nothing changes.
03
Agent
Execute. The AI writes, edits, and runs code across your project. This is where things get built.
The rule: Always start in Plan. Resist the urge to jump straight into Agent.

// 4:30 — ARCHITECTURE
Architecture made simple
Just enough to be dangerous:
Static hosting
GitHub Pages
HTML/CSS/JS files served directly. No server. Free. Perfect for tools and landing pages.
App hosting
Vercel
Runs server-side code. Handles routes, APIs, dynamic content. Still one-push deploy.
The difference
Client vs. Server
Client = runs in browser. Server = runs on someone else's computer. "Deploy" = putting code on that computer.
